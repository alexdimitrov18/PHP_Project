User login functionality (PHP/MySQL) Login functionality can be accessed through login.php. The form requires both username and password to work. The login is done with the password_verify function which hashes and salts the plain text input of the user and compares it to the hashed and salted password in the database (table users). The navigation bar changes whenever a user logs in (Login changes to Logout and the username appears as a navbar link leading to profile.php) Logout.php destroys the current session, which logs out the user and forwards them to login.php. login.php logout.php
Ability to select, add, edit and delete information from a database (PHP/MySQL) Such functions were implemented in the profile.php page, which is accessible only after a user has logged in. Four buttons were added to the same form and each was accessed with the "array_key_exists" function. The submission button requires data in all fields marked with a red star. Delete button directly deletes all data for the specific user. Show Data selects the data from the database and displays it to the right of the form. Update allows the user to update the info for only one cell of the table depending on which field had any user input. All of the data is stored in the database (Table userInfo) based on the user's username. The username stored in the session is the same as the ID column in the table, which allows access/submission of data only for the specified user. profile.php
Appropriate consideration of relevant laws A Privacy policy page was added with a custom GDPR template. A jQuery cookie plugin was used to gather cookies after the user's consent. (Cookie notification appears in index.php) cookies.js jquery.ihavecookies.min.js privacy_policy.php
SQL queries should be written as prepared statements All SQL queries were changed to prepared statements in both profile.php and login.php. login.php profile.php
Passwords should be salted and hashed Passwords stored in the database (Table users) were pre-hashed and salted using the default php password_hash function. Then they are verified with the use of password_verify. login.php
Validation of user input Both login.php and profile.php have user input validation for all input fields. A function takes the input then implements the trim, stripslashes and htmlspecialchars functions then returns the result. Said result is used in prepared statements and stored in the database (Table userInfo) if it's personal data of the user. login.php profile.php
